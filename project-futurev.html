FutureV Storefront

.NET 9 ASP.NET Core MVC PostgreSQL Docker Clean Architecture

Architected and built a high-performance, scalable eCommerce platform for futuristic vehicles using .NET 9 and Clean Architecture. Engineered a robust backend featuring ASP.NET Core Identity for secure role-based access, the Repository Pattern for decoupled data access, and FluentValidation for strict data integrity.

View Repository

Technical Architecture

Clean Architecture

Refactored a monolithic MVC application into a strictly layered solution (Core, Data, App) to enforce separation of concerns. Decoupled business logic from infrastructure for long-term maintainability.

Repository Pattern

Abstracted Entity Framework Core logic behind interfaces, allowing for flexible data source management and easier unit testing without tight coupling to the database context.

Secure Identity

Integrated ASP.NET Core Identity to replace basic authentication, implementing secure, role-based access control for the admin dashboard with cookie-based sessions.

FluentValidation

Replaced standard DataAnnotations with FluentValidation to implement complex, strongly-typed validation rules, keeping domain models clean and controller logic focused.

Problem Solving

The Challenge

The initial application was tightly coupled to the database context, making unit testing difficult and refactoring risky. Additionally, managing validation for complex vehicle data was cluttering controller logic.

The Solution

I implemented the Repository Pattern to invert dependencies, allowing the Application layer to depend on abstractions. I also integrated FluentValidation to encapsulate rules in dedicated classes, removing validation logic from models.
